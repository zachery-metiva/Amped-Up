Amped Up: Application Workflow Architecture

Field evidence to AI validation, predictive risk, and real-time utility operations dashboard
amped-up.online

Data Collection

The field layer

01
Community & Utility Employee
Photo Capture

Community members and utility workers capture pole structure integrity, GPS position, and more on-site detailed descriptions near real time.

02
Pole and map data

DTE/OSM pole data supplies asset location, circuit, owner, classification, and local context.

03
Risk signals

Environmental and asset indicators feed vegetation, flood, soil, age, lean, and inspection risk.

Cloud Processing

The AI and API engine

API
Render FastAPI service

REST routes handle dashboard state, report submission, notes, status updates, photos, and WebSocket events.

DB
PostgreSQL

Stores poles, reports, photos, users, history, violation rules, notes, and predicted_reports rows.

AI
Watsonx + risk engine

AI validates utility photos while daily cron computes predictive risk and severity bands.

Data Consumption

The operations interface

WEB
React and Vite

The dashboard renders map views, submitted reports, predicted issues, pole details, photos, and review actions.

LIVE
Operations dashboard

Operators review community reports, assign severity, approve or dismiss work, add notes, and track live KPIs.

RISK
Predicted issue view

Switch shows predicted reports and risk strip totals sourced from the daily risk table.
